```html
function updateStats(jsonString, parsedObject) {
    const size = new Blob([jsonString]).size;
    const lines = jsonString.split('\n').length;
    const depth = calculateDepth(parsedObject);
    
    document.getElementById('jf-size').textContent = `${size} B`;
    document.getElementById('jf-lines').textContent = lines;
    document.getElementById('jf-depth').textContent = depth;

function calculateDepth(obj, currentDepth = 0) {
    if (typeof obj !== 'object' || obj === null) return currentDepth;
    
    let maxDepth = currentDepth;
    for (const key in obj) {
        if (obj.hasOwnProperty(key)) {
            const depth = calculateDepth(obj[key], currentDepth + 1);
            maxDepth = Math.max(maxDepth, depth);
        }
    }
    return maxDepth;
}
```

## **üéØ THE DESIGN QUALITY CHECKLIST**

### **Visual Consistency Audit (Run Before Every Deploy)**
1. **Spacing**: All elements use spacing scale (8px √ó œÜ)
2. **Typography**: Font sizes follow type scale (16px √ó œÜ‚Åø)
3. **Color**: All colors from defined palette, no hex values
4. **Border Radius**: Consistent radius scale applied
5. **Shadows**: Correct shadow layers for depth hierarchy
6. **Transitions**: All animations use standard timing (300ms, ease-cubic)
7. **Responsive**: Works on 320px ‚Üí 2560px screens
8. **Accessibility**: Color contrast > 4.5:1, keyboard navigable
9. **Performance**: No layout shifts, <100ms interaction response
10. **Print**: All tools print cleanly with no UI elements

### **The Golden Ratio Quick Test**
For any element, calculate:
```
width √∑ height ‚âà 1.618
OR
height √∑ width ‚âà 1.618
```
If not within 5% tolerance, redesign.

### **The Three-Second Rule**
A new user should understand:
1. **What it is** (in 1 second)
2. **How to use it** (in 2 seconds)  
3. **What value it provides** (in 3 seconds)

If any fails, simplify.

## **üèÜ THE DESIGN EXCELLENCE MANIFESTO**

> **"Great design is invisible. It doesn't shout about how beautiful it is‚Äîit quietly makes everything work better. Every decision must answer: Does this make the tool more useful? Does this make the interface more understandable? Does this make the experience more enjoyable? If not, remove it. The most beautiful interface is the one that disappears, leaving only pure utility in its wake."**

### **The Three Design Virtues**
1. **Restraint** - The courage to remove what isn't essential
2. **Clarity** - The commitment to make everything understandable  
3. **Empathy** - The wisdom to design for all humans, not just some

### **The Designer's Oath**
